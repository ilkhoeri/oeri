## Data Types

**Determines Data:**

- Define data by default using `props.actions` or manually specifying the children component.
- Either `props.actions` or `children` of `<Command>` must be specified, you cannot use both at the same time or omit both.
- Using both at the same time will confuse the function and omit both will not do anything.
- To prevent this we use error throwing so that the function can run properly.
- Another way is to use the `<Command.Root>` compound component as the parent of another `<Command>` compound.
